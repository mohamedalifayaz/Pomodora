🍅 Pomodoro Timer — Web App

A fully functional, visually polished Pomodoro productivity timer. No frameworks, no build steps — pure HTML/CSS/JS served directly from index.html.

Technology Stack

HTML5 — SVG ring + Google Fonts CSS3 — custom properties, animations Vanilla JS ES6+ — Web Audio API, localStorage

app.js — Section Architecture

All logic lives in app.js, organised by // ─── Section Name ─── banners:

Key Constants

CIRCUMFERENCE         = 2π × 96  // ≈ 603.19 (SVG circle r=96)

SVG Progress Ring

Defined in index.html as inline <svg viewBox="0 0 220 220">.

  • 🔘 Track: <circle class="ring-track"> — static grey arc
  • 🔴 Progress: <circle class="ring-progress" id="ringProgress"> — animated
  • 🎨 Gradients: ringGradient (red) · breakGradient (green) · warnGradient (orange)
  • ✨ Filter: <filter id="glow"> — Gaussian blur + merge for glow effect
  • 📐 Formula: offset = CIRCUMFERENCE × (1 − ratio)

🔔 Audio System

Built entirely on the Web Audio API — no audio files needed. AudioContext is lazily instantiated and reused (singleton pattern) to respect browser autoplay policies.

Bell Sound (playBell)

Four sine-wave oscillators at harmonic frequencies with exponential gain decay, repeated 3 times with 0.7 s spacing.

520 Hz 1 040 Hz 1 560 Hz 2 600 Hz

Called once when the timer starts and once when it ends.

⚠️
Always wrap Web Audio API calls in try/catch and fail silently — browser policy differences can throw exceptions. The old playTick / startTicking / stopTicking functions have been removed.
🐍 Python CLI Games

Three beginner-friendly text-based games. Standalone scripts, standard library only — no requirements.txt needed.

guess_the_number.py

Entry: guess_the_number()

Custom range input → random secret → 10 attempts → hot/cold feedback → win streak → play-again loop. Catches ValueError for non-integer input.

hangman.py

Entry: hangman() · Helpers: choose_random_word(), display_word()

Random word from hardcoded list → letter guessing → 6 incorrect guesses allowed → masked word each round.
Word list: ["python", "hangman", "programming", "challenge", "computer"]

play_tic_tac_toe.py

Entry: play_tic_tac_toe() · Helpers: print_board(), check_win(), is_full()

Two-player 3×3 grid as a nested list. Row/column input, win check (rows, cols, diagonals), draw check. Empty cell = " ".

🎨 Coding Conventions

JavaScript (app.js)

  • ✅ No framework, no transpiler — plain ES6+ runs directly in the browser
  • ✅ Single state object pattern — all mutable data in one place
  • ✅ DOM refs declared once at the top, never re-queried inside functions
  • ✅ Section banners: // ─── Section Name ───
  • ✅ Always wrap Web Audio calls in try/catch and fail silently
  • ✅ Use pomo_ prefix for all localStorage keys
  • ✅ Prefer const / let; never var. Semicolons are used throughout.

CSS (style.css)

  • ✅ Use CSS custom properties (var(--name)) for every repeated value
  • ✅ Section-per-component comment banner pattern
  • ✅ @keyframes at the bottom of their relevant section
  • ✅ Body-class-driven theming instead of inline styles

Python (game files)

  • ✅ PEP 8 style — snake_case for functions and variables
  • ✅ Each game is a single self-contained function — no classes needed
  • ✅ if __name__ == "__main__" guard on every file
  • ✅ Do not remove enhancement task lists — they are part of the educational design
🚧 Known Issues / Notes
#1
Duplicate <div class="timer-card"> in index.html — around lines 37–40. Browser renders correctly due to error recovery, but should be cleaned up.
#2
No package.json / build tooling — zero dependencies, zero build steps. Do not add a bundler unless explicitly asked.
#3
Python games have no unit tests — rely on manual CLI interaction. If tests are needed, use unittest with unittest.mock.patch to mock input().
#4
No long break mode — only focus and short break are implemented. Adding it requires a new key in CONFIG, a new gradient, and logic in handleTimerEnd.
#5
Daily stats reset — pomo_today is cleared when new Date().toDateString() differs from pomo_date. Runs at page load only.
#6
Audio requires user gesture — AudioContext starts suspended. getAudioCtx() handles .resume() automatically; bell only plays after the first Start click.

🤖 Instructions for AI Agents
📝
Before modifying app.js: Check the section banners and insert new code inside the appropriate section. Preserve the // ─── ... ─── banner style.
🎨
Before modifying style.css: Check if a CSS variable already exists for the colour/value before adding raw values.
💾
Before adding features to the Pomodoro timer: Check if the state object needs a new field and if it should be persisted to localStorage.
🐍
For Python games: Keep each game self-contained. Do not introduce imports beyond the Python standard library without explicit request.
🚫
Do not introduce a build system (webpack, vite, etc.) unless explicitly requested.
🌐
The web app runs without a server — keep it that way unless instructed otherwise.
🐛
Fix the duplicate <div class="timer-card"> in index.html if doing any HTML edits — it is a pre-existing bug.
